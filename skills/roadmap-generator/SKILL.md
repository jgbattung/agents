# Roadmap Generator Skill

Generates a `ROADMAP.md` (and optionally `ROADMAP.html`) in the `backlog/` folder that provides an epic-level view of project progress.

## When to Run

- After the PM Agent creates or updates backlog items
- After the Integrator archives a completed item
- When the user explicitly requests a roadmap refresh

## How to Generate ROADMAP.md

### Step 1: Collect All Items

Read all `.md` files from both `backlog/` and `backlog/archive/` (excluding `ROADMAP.md` and `ROADMAP.html`). Parse YAML frontmatter from each.

### Step 2: Group by Epic

Group items by their `epic` field. Sort epics alphabetically. Within each epic, sort items by ID (ascending).

### Step 3: Calculate Progress

For each epic:
- Count total items
- Count items with `status: done`
- Calculate percentage: `done / total`

### Step 4: Generate Progress Bar

Use block characters to create a 10-segment progress bar:

- Full block: `█` (U+2588)
- Empty block: `░` (U+2591)

Formula: filled segments = `round(done / total * 10)`

Example: 4/10 done → `[████░░░░░░] 4/10 stories`

### Step 5: Write ROADMAP.md

Use this exact structure:

```markdown
# Roadmap — {Project Name}

> Last updated: {YYYY-MM-DD}

## {Epic Name (title-cased from slug)}
[████░░░░░░] {done}/{total} stories

| ID | Title | Priority | Status |
|---|---|---|---|
| {ID} | {Title} | {Priority or —} | {Status} |
| ... | ... | ... | ... |

## {Next Epic}
...

---

## Summary
- **Total stories:** {N}
- **Done:** {N} ({percentage}%)
- **In progress:** {N}
- **Ready:** {N}
- **Draft:** {N}

---
*Auto-generated by roadmap-generator skill.*
```

### Formatting Rules

- Epic names: Convert kebab-case slug to Title Case (e.g., `transaction-management` → `Transaction Management`)
- Items with `status: done` should show in the table but are visually distinguished — no special formatting needed since the status column makes it clear
- Priority column: Show `—` if priority is not set
- Order within epic table: done items first, then in-progress, then ready, then draft

## ROADMAP.html Generation

After writing `ROADMAP.md`, also generate a `ROADMAP.html` file that renders a visual kanban-style board. This is a single self-contained HTML file (inline CSS, no external dependencies) that can be opened directly in a browser.

### HTML Board Structure

```
┌─────────────┬─────────────┬─────────────┬─────────────┐
│   Draft     │   Ready     │ In Progress │    Done     │
├─────────────┼─────────────┼─────────────┼─────────────┤
│ [card]      │ [card]      │ [card]      │ [card]      │
│ [card]      │ [card]      │             │ [card]      │
│             │             │             │ [card]      │
└─────────────┴─────────────┴─────────────┴─────────────┘
```

Each card should display:
- ID and title (bold)
- Epic (as a colored tag — assign a consistent color per epic)
- Priority (as a badge: P0=red, P1=orange, P2=blue, P3=gray)

### HTML Design Requirements

- Clean, modern design with a light background
- Responsive — works on different screen widths
- Cards are grouped into four columns by status
- Each column has a header with the count of items
- Epic tags use consistent colors (hash the epic slug to derive a hue)
- Include a summary bar at the top with overall progress
- No JavaScript required — pure HTML + CSS
- File should be self-contained and under 50KB
